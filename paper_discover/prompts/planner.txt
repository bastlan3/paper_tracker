## Instructions

Produce a structured plan with these elements:

1. **intent**: A single clear sentence stating exactly what the search is trying to find.

2. **concepts**: 5–15 key concepts/terms that define the topic. These will be used for query generation.

3. **dimensions**: Decompose the intent into orthogonal searchable dimensions.
   For each dimension specify:
   - name: short identifier (e.g. "intervention", "population", "outcome")
   - value: what we are looking for in that dimension
   - essential: true if a paper missing this dimension should not be CORE
   - critical: true if a paper missing this dimension should NEVER pass Gate B
     (use sparingly — only for truly mandatory constraints)

4. **scope**: Optional filters. Include only what the user explicitly stated or strongly implied.
   - date_from: earliest publication year (YYYY-MM-DD)
   - date_to: latest publication year (or omit for "now")
   - languages: list of ISO 639-1 codes (default ["en"])
   - exclude_study_types: e.g. ["case_report", "editorial", "news"]

5. **anchors**: List of paper identifiers already provided (DOIs, arXiv IDs). Preserve as-is.

6. **depth**: One of "fast" / "standard" / "deep" / "unlimited". Infer from context; default "deep".

Output ONLY valid JSON matching the schema. No prose, no markdown fences.
